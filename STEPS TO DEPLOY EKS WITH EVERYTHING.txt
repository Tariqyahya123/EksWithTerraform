STEPS TO DEPLOY EKS WITH EVERYTHING CONFIGURED:

- Pull the git repo

- enter the name of the key pair in the bastionhost file.


- terraform apply inside the EksWithTerraform directory.


- note the output of the terraform apply command which contains the arn's of the two required IAM roles for the autoscaler and the aws-load-balancer-controller addons.

- install aws-load-balancer helm upgrade -i aws-load-balancer-controller ./K8s-deployment-files/aws-load-balancer-controller -n kube-system  --set clusterName=my-cluster --set serviceAccount.annotations.eks\\.amazonaws\\.com\\/role-arn= arn:aws:iam::411310121956:role/eks-ingress-role --set serviceAccount.name=aws-load-balancer-controller 


- install eks-auto-scaler helm upgrade -i eks-auto-scaler ./K8s-deployment-files/eks-auto-scaler -n kube-system --set autoScalerRoleArn=arn:aws:iam::411310121956:role/cluster-auto-scaler-role


- install wordpress helm upgrade -i wordpress-and-mysql ./K8s-deployment-files/wordpress-deployment -n default
